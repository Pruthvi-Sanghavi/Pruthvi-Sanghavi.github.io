Introduction

Optimization of Autonomous Drone Delivery Systems/shared electric scooter/Vehicle fleet operations has become essential for the fleet owners to increase their revenue. Ride revenue can be increased through the increase in the number of deliveries/rides per battery charge. One approach that is proposed in this project to achieve more scooter rides per charge is the concept of pig tailing. An autonomous Drone/Electric vehicle can use the transit networks to move to the location of its request. The agent can sit on the transit or get inside the transit to save battery and reach the destination safely.

The map shown here is a small area in Washington DC. Red triangles here represent scooter depots. Red path represent the transit routes and black paths indicate the path of the autonomous drone/electric scooter.

Assumptions

  1. The number of drones/scooters/robots and the depots are constant.
  2. Transit networks are deterministic with respect to the locations and vehicle travel time.
  3. There is a capacity of transit networks to carry these scooters.
  4. Transit networks are unaffected because of this robot pigtailing.
  5. Drones/Electric scooters can be recharged in a negligible amount of time using swappable batteries.
  6. Depots have unlimited drone/scooter capacity.

Cases

  1. When a customer calls an drone/electric scooter, the drone/scooter reaches there from the depot using pigtailing to optimize some cost.
  2. When a customer drops off the scooter after a ride, the scooter depending upon the charge it has should decide how to reach the depot.
  3. When the customer calls it not from the depot should it pigtail or go on its own picking up someone on the way increasing ride revenue.

Problem Formulation

Let the problem be formulated as follows:

Size of the Drone/scooter fleet = m

Number of depots = l with the geographical locations given as Vd:= {d1, d2, d3,...,dl}.

Number of ride requests = k from the geographical locations given as Vc:= {c1, c2, c3,...,ck}.

where k>>m.

Datasets

For this project GBFS (General Bikeshare Feed Specification) dataset was used to know the station positions of the bikes and electric scooter in a small area in Washigton DC. Link to the GBFS website. The station .json consists of the information about the latitude and the loongitude of the bike stations. Transit Data can be extracted from the GTFS Dataset which provides the file with the route coordinates of a transit vehicle. These data files are very large and a lot of computer processing went into it. So for this project, a toy data was created for the transit network route for the purpose of demonstration. The map is generated using a shape file of the road networks in washington DC downloaded from the Washington DC open source datasets.

Results

This project is not a strict implementation of the above research paper. The results generated in this project are some of the work towards my final project. This project is focussed towards loading the GBFS and GTFS datasets while also formulating the TSP algorithms while relieving some of the constraints and finding the shortest path from depots to the scooters and also the interdepot travel sequence.
